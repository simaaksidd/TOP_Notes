Introduction

Basic syntax includes a selector and property:value pair

Selectors

Universal selector: *

Type Selectors are for html tags like "div"

selectors select every element in the given class

Elements can have multiple classes with whitespace to separate them

ID selectors are similar to class selectors but the element can only have ONE ID

you can group similar property:value pairs together

Chaining selectors is useful when you have multiple class selectors inside a container tag. You can also do this with ID tags

Chaining type selectors does not work because a tag cannot have more than one type, i.e. a div cannot be a div and a p at the same time

The Cascade

Sometimes CSS doesn't do what we want it to do. This may be because of three things: personal error, default browser styles, and the Cascade. The cascade is what determines which rules actually get applied to our HTML. There are different factors that the cascade uses to determine this.

Specificity

Sometimes an element will have multiple, conflicting declarations assigned to it, because of this, CSS has an order of Specificity. The most specific CSS declaration is what the element will go with. Inline styles are the most specific, then all of the selectors we have mentioned follow in this order:

  1. ID selectors (most specific)
  2. Class selectors
  3. Type selectors

    4. in the case that the element has an element that shares multiple declarations with the same level of specificity, then the one with more of that selector will take precidence. Ex: 2 class selectors takes precidence over one class selector
